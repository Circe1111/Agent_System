## 1. 环境准备与依赖安装

### 1.1 前置条件

| 组件 | 版本要求 | 用途 |
|------|----------|------|
| Python | ≥ 3.11 | 后端运行环境 |
| Node.js | ≥ 22.18.0 | 前端运行环境 |
| npm | ≥ 10.x | 前端包管理 |
| Git | 任意版本 | 版本控制 |

### 1.2 安装 Python

**Windows:**
1. 从 https://www.python.org/downloads/ 下载 Python 3.11+ 安装包
2. 安装时务必勾选 **"Add Python to PATH"**
3. 验证安装：
   ```powershell
   python --version
   pip --version
   ```

### 1.3 安装 Node.js

**Windows:**
1. 从 https://nodejs.org/ 下载 Node.js 22 LTS 版本
2. 安装时保持默认选项即可
3. 验证安装：
   ```powershell
   node --version
   npm --version
   ```

### 1.4 配置 Python 虚拟环境

在项目根目录下操作：

```powershell
# 进入项目目录
cd E:\学校\大3\大三下\B-软件综合实践实验\Edu_Code\Agent_System

# 创建虚拟环境（在项目根目录下）
python -m venv venv

# 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 如果遇到 PowerShell 执行策略错误，先执行：
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

激活后，终端提示符前会出现 `(venv)`。

### 1.5 安装后端依赖

```powershell

pip install --upgrade pip
pip install -r backend\requirements.txt
```

如果遇到 `faiss-cpu` 安装失败，可使用以下替代方案：

```powershell

# 或
pip install numpy && pip install faiss-cpu
```

### 1.6 安装前端依赖

```powershell

cd frontend

# 安装依赖
npm install

# 返回项目根目录

```

## 2. 项目本地配置

### 2.1 配置环境变量

```powershell
# 复制环境变量模板
copy .env.example .env
```

编辑 `.env` 文件，配置以下关键项：

```ini
# === LLM 配置（必填）===
LLM_API_KEY="sk-你的API密钥"
LLM_BASE_URL="https://api.openai.com/v1"
LLM_MODEL="gpt-4o"

# === JWT 密钥（必改）===
JWT_SECRET_KEY="生成一个32位以上的随机字符串"

# === 数据库配置（默认 SQLite，零配置即可用）===
DATABASE_URL="sqlite:///./edu_agent.db"

# === 端口配置 ===
BACKEND_PORT=8000
FRONTEND_PORT=80
```

> **注意**：SQLite 是项目默认数据库，无需额外配置即可开发。如需切换 MySQL，将 `DATABASE_URL` 改为 `mysql+pymysql://user:password@localhost:3306/dbname`。

### 2.2 初始化数据库和种子数据

```powershell
# 确保在项目根目录，虚拟环境已激活

# 方法一：直接运行种子脚本（推荐）
cd backend
python scripts/seed_db.py

# 预期输出：
# Created admin user: admin/admin123
# Created student user: student1/123456
# Database seeded successfully!

# 方法二：如果 FAISS 向量数据库需要重建
python scripts/rebuild_faiss.py
```

### 2.3 运行数据库迁移（可选）

如果数据库表结构有变更，执行 Alembic 迁移：

```powershell
cd backend
alembic upgrade head
```

### 2.4 验证数据库

```powershell
# 测试数据库连接
python -c "from database.connect import get_db; next(get_db()); print('✅ 数据库连接正常')"
```

**预期结果**：项目根目录下生成 `edu_agent.db` 文件（SQLite），包含 `edu_user`、`students` 等数据表。

## 3. 启动开发服务器

### 3.1 启动后端服务（终端 1）

```powershell
# 确保在项目根目录，虚拟环境已激活

# 直接启动（支持热重载）
cd backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**说明：**
- `--reload` 参数启用热重载，修改代码后自动重启
- 如遇端口被占用，修改 `.env` 中的 `BACKEND_PORT` 或使用 `--port 8001`

**验证后端启动：**
打开浏览器访问 http://localhost:8000/docs，应看到 Swagger API 文档页面。
访问 http://localhost:8000/health，应返回 `{"status":"ok"}`。

### 3.2 启动前端服务（终端 2）

```powershell
# 新开一个 PowerShell 窗口
cd E:\学校\大3\大三下\B-软件综合实践实验\Edu_Code\Agent_System\frontend

# 启动 Vite 开发服务器
npm run dev
```

**说明：**
- Vite 默认运行在 http://localhost:5173
- 支持 HMR（热模块替换），修改前端代码后浏览器自动刷新
- 前端通过 `VITE_API_BASE_URL=http://127.0.0.1:8000` 连接后端（配置在 `frontend/.env.development`）

**验证前端启动：**
打开浏览器访问 http://localhost:5173，应看到登录页面。

```python
"""测试 CRUD 操作中的密码工具函数"""
import pytest
from database.crud import hash_password, verify_password

class TestPasswordUtils:
    def test_hash_password_returns_string(self):
        """测试 hash_password 返回字符串"""
        hashed = hash_password("test_password")
        assert isinstance(hashed, str)
        assert len(hashed) > 0

    def test_verify_password_correct(self):
        """测试正确密码验证通过"""
        hashed = hash_password("test_password")
        assert verify_password("test_password", hashed) is True

    def test_verify_password_incorrect(self):
        """测试错误密码验证不通过"""
        hashed = hash_password("test_password")
        assert verify_password("wrong_password", hashed) is False

    def test_verify_password_empty(self):
        """测试空密码验证"""
        hashed = hash_password("test_password")
        assert verify_password("", hashed) is False

    def test_hash_different_for_same_password(self):
        """测试相同密码每次哈希结果不同（bcrypt 加盐特性）"""
        password = "same_password"
        hash1 = hash_password(password)
        hash2 = hash_password(password)
        assert hash1 != hash2
        assert verify_password(password, hash1) is True
        assert verify_password(password, hash2) is True
```

```python
class TestUserFlow:
```

```python
import os

sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..', '..'))

from locust import HttpUser, task, between
import json

class EduAgentUser(HttpUser):
    """模拟用户行为"""

        response = self.client.post(

        )
        data = response.json()
        self.token = data.get("access_token") or data.get("token", "")
        self.headers = {"Authorization": f"Bearer {self.token}"}

    @task(3)
    def health_check(self):
        """健康检查（高频）"""
        self.client.get("/health")

    @task(2)
    def get_user_info(self):

    @task(1)

        self.client.get("/resources/")

    @task(1)
```

## 9. 常见问题排查

### Q1：`pip install -r requirements.txt` 安装失败

**原因**：`faiss-cpu` 或 `bcrypt` 在 Windows 上编译困难。

**解决方案**：
```powershell
# 先安装 numpy
pip install numpy

# 单独安装 faiss-cpu
pip install faiss-cpu --no-build-isolation

# 如果仍然失败，尝试预编译版本
pip install faiss-cpu --only-binary=faiss-cpu

# bcrypt 安装失败时
pip install bcrypt --no-build-isolation
```

### Q2：后端启动报 `ModuleNotFoundError`

**原因**：Python 导入路径不正确。

**解决方案**：
```powershell
# 确保在 backend 目录下运行 uvicorn
cd backend

# 手动设置 PYTHONPATH
$env:PYTHONPATH = "E:\学校\大3\大三下\B-软件综合实践实验\Edu_Code\Agent_System\backend"
uvicorn main:app --reload
```

### Q3：前端启动报 `Error: Cannot find module`

**原因**：npm 依赖未正确安装。

**解决方案**：
```powershell
cd frontend
rm -r node_modules  # 删除 node_modules
del package-lock.json
npm install
```

### Q4：前端登录时提示 CORS 错误

**原因**：前端和后端端口不匹配，或 `allow_origins` 未配置。

**解决方案**：
1. 检查 `backend/main.py` 中的 `allow_origins` 是否包含前端地址
2. 检查 `frontend/.env.development` 中的 `VITE_API_BASE_URL` 是否正确
3. 默认 Vite 运行在 5173 端口，后端 CORS 已配置 5173-5176

### Q5：SQLite 数据库锁定错误

**原因**：多个进程同时写 SQLite 数据库。

**解决方案**：
```powershell
# 停止所有后端进程，删除数据库文件重新初始化
cd backend
deactivate  # 退出虚拟环境
# 手动关闭所有 Python 进程
taskkill /F /IM python.exe

# 重新启动
.\venv\Scripts\Activate.ps1
del edu_agent.db
python scripts/seed_db.py
uvicorn main:app --reload
```

### Q6：AI 聊天没有响应

**原因**：LLM API Key 无效或网络不通。

**解决方案**：
1. 检查 `.env` 中的 `LLM_API_KEY` 是否正确
2. 使用 curl 测试 API 连通性：
   ```powershell
   curl -X POST "https://api.openai.com/v1/chat/completions" `
        -H "Authorization: Bearer sk-your-key" `
        -H "Content-Type: application/json" `
        -d '{"model":"gpt-4o","messages":[{"role":"user","content":"hello"}]}'
   ```
3. 检查后端日志：`uvicorn` 终端输出的错误信息

### Q7：测试报 `database is locked`

**原因**：SQLite 测试数据库冲突。

**解决方案**：在 `conftest.py` 中使用内存数据库：

```python
# 将 TEST_DATABASE_URL 改为内存模式
TEST_DATABASE_URL = "sqlite://"
# 并在 create_engine 中添加 connect_args={"check_same_thread": False}
```
