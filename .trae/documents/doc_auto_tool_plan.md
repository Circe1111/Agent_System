# 文档自动化工具开发计划

## 项目概述

基于现有测试报告生成脚本，提取并重构为一个可复用、可扩展的**文档自动化工具**（DocAutoTool）。该工具将提供：

1. **终端输出自动截图**：根据测试脚本终端运行结果，自动生成终端风格截图（支持白色/黑色主题）
2. **文档模板解析与生成**：解析参考报告文档和当前文档，生成符合模板格式的最终文档
3. **高扩展性架构**：支持后续扩展其他文档格式和功能

## 一、需求分析

### 1.1 功能需求

| 需求编号 | 功能描述 | 详细说明 |
|---------|---------|---------|
| FR-01 | 终端输出截图 | 根据测试脚本输出，自动生成终端风格截图，支持白色/黑色主题切换 |
| FR-02 | 截图嵌入文档 | 将自动生成的截图嵌入到最终报告文档中 |
| FR-03 | 参考文档解析 | 解析参考报告文档（docx），提取格式模板（标题、表格、图片位置等） |
| FR-04 | 内容合并生成 | 将当前文档内容按照参考文档格式模板进行合并，生成最终文档 |
| FR-05 | 持续扩展能力 | 工具架构支持后续扩展其他文档格式（pdf、html等）和新功能 |

### 1.2 技术需求

| 技术点 | 需求说明 |
|-------|---------|
| 截图方案 | 除 Pillow 外，需提供其他稳定方案（如 `mss`、`pyautogui`、终端渲染方案） |
| 文档解析 | 使用 `python-docx` 解析 docx 文档结构 |
| 可扩展性 | 采用插件化/模块化设计，便于后续扩展 |
| 稳定性 | 确保在 Windows 环境下稳定运行 |

### 1.3 非功能需求

- **轻量级**：工具本身代码量控制在合理范围内
- **易用性**：提供清晰的命令行接口和操作文档
- **兼容性**：支持 Python 3.10+，Windows 环境

## 三、技术方案

### 3.1 终端截图方案对比

| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
|-----|------|------|------|---------|
| Pillow | 直接渲染文本到图像 | 无需实际终端窗口，可控性强 | 需要字体配置，中文支持需额外处理 | 性能测试脚本自动截图 |
| mss | 截取屏幕指定区域 | 真实终端效果，支持窗口定位 | 需要终端窗口在前台，依赖系统窗口管理 | 交互式测试截图 |
| pyautogui | 模拟鼠标操作+截图 | 功能全面，支持窗口截图 | 速度较慢，依赖屏幕分辨率 | 复杂场景截图 |
| rich + svg | 使用rich渲染为SVG再转PNG | 终端样式丰富，可精确控制 | 需要额外依赖转换工具 | 高质量终端输出截图 |

**推荐方案**：采用**策略模式**，默认使用 Pillow 渲染（无需窗口），同时支持 mss 作为备选方案。

### 3.2 文档解析方案

使用 `python-docx` 解析参考文档，提取以下结构信息：

1. **标题层级**：提取所有 heading 元素及其层级
2. **表格定义**：提取表格结构（列数、行数、表头样式）
3. **图片占位符**：识别图片位置和尺寸
4. **段落样式**：提取字体、字号、对齐方式等

### 3.3 扩展性设计

采用**插件化架构**，通过适配器模式实现：

```python
# 截图策略接口
class ScreenshotStrategy(ABC):
    @abstractmethod
    def capture(self, content, theme='light') -> str:
        pass

# 文档适配器接口
class DocumentAdapter(ABC):
    @abstractmethod
    def parse(self, filepath):
        pass
    
    @abstractmethod
    def generate(self, structure, content):
        pass
```

## 五、实施步骤

### 步骤 1：创建工具目录结构

创建 `tools/doc_auto_tool/` 目录及子模块结构。

- 实现 `terminal_renderer.py`，支持白色/黑色主题

### 步骤 4：实现文档生成器

- 实现 `document_generator.py`，根据模板合并内容
- 支持截图自动嵌入

### 步骤 5：实现命令行接口

- 实现 `cli.py`，提供命令行操作接口
- 支持参数：输入文件、输出文件、主题选择等

### 步骤 6：编写使用文档

- 编写 `README.md`，包含安装、配置、使用方法

### 步骤 7：测试验证

- 测试终端截图功能（白色/黑色主题）
- 测试文档解析与生成功能
- 验证完整工作流程

## 七、风险与应对

| 风险 | 应对措施 |
|-----|---------|
| Pillow 中文渲染问题 | 预设多个中文字体路径，自动检测可用字体 |
| mss 窗口定位失败 | 提供窗口标题匹配、坐标手动指定等多种定位方式 |
| docx 格式解析不完整 | 优先提取核心结构（标题、表格、图片），复杂样式保留原始格式 |
| 扩展性不足 | 采用适配器模式，预留接口供后续扩展 |
